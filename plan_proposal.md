# 全体計画
> パッケージ横断の話だけ書く。個別の実装詳細は `packages/<pkg>/plan.md` へ。

## 前提・制約

- 対象: 今後立ち上げる個人プロジェクト全般
- ロールバック: 適用前に対象リポジトリで `git stash` または作業ブランチを切る

---

## Why（なぜ作るか）
> 変更する場合は `docs/decisions/` に理由を記録してから更新する。

- プロジェクトごとにディレクトリ構造がバラバラで、Claude Code が文脈を掴むまでに時間がかかる
- 均一な構造があれば、新規プロジェクトでも即日 Claude Code が機能する状態になる
- 参考: [`モノレポ案.md`](./モノレポ案.md)、[`root/`](./root/)（サンプル実装）

## What（何を達成するか）
- 新規プロジェクトを立ち上げたとき、コマンド1つでモノレポ構造が生成できる
- パッケージを追加するとき、コマンド1つでテンプレート通りの構造が追加できる
- Claude Code がどのプロジェクトでも「正しい場所」に出力できる

## How（どうやって実現するか）

Claude Code のスキル（`~/.claude/skills/`）を中心に実現する。各スキルはグローバルに配置し、どのプロジェクトでも共用できる。

### 必要スキル一覧

| スキル | トリガー | 概要 |
|---|---|---|
| `monorepo-init` | `/monorepo-init` | 新規プロジェクトに `root/` のテンプレート構造を展開する |
| `package-add` | `/package-add` | 既存モノレポに新しいパッケージをテンプレートから追加する |
| `template-sync` | `/template-sync` | テンプレート（`root/`）の変更を既存プロジェクトに同期する |
| `critique` | `/critique` | 対象ファイルを別インスタンスで批判し `_review.md` を生成する |
| `propose` | `/propose` | `_review.md` をもとに修正案を作成し `_proposal.md` を生成する |

> 採用ループ（`/critique` → `/propose` → 比較・採用）の詳細は `decisions/0001-提案の採用ループ.md` を参照。

### スキル詳細

#### `/monorepo-init`
- **目的**: 新規プロジェクトでモノレポ構造をゼロから作る
- **処理**: `degit hirayama-yuuichi/claude-code-monorepo-template/root` で `root/` の内容を取得し、プロジェクト名等のプレースホルダを対話的に置換して展開
- **README.md**: 展開後の `README.md` 冒頭にテンプレートリポジトリへの参照行を自動で埋め込む（人間・Claude 両方が読む入口として機能させる）
- **エラー**: 既存ファイルがある場合は上書き確認を求めて中止

#### `/package-add`
- **目的**: モノレポに新しいパッケージを追加する
- **処理**: `root/packages/package-a/` をテンプレートとしてコピーし、パッケージ名を対話的に置換
- **エラー**: 同名パッケージが既存の場合は中止

#### `/template-sync`
- **目的**: テンプレート更新を既存プロジェクトへ伝搬する
- **処理**: `root/` の現在の状態と対象プロジェクトの差分を表示し、確認後に適用
- **エラー**: 既存ファイルは上書きせずマージ候補として提示する

#### `/critique`
- **目的**: Claude 提案を別インスタンスで批判し、バイアスなしの評価を得る
- **処理**: 指定ファイルを別インスタンスに渡して批判させ、結果を `<対象>_review.md` に保存

#### `/propose`
- **目的**: 批判結果をもとに修正案を作成する
- **処理**: `<対象>_review.md` を読み込み、元ファイルに反映した修正案を `<対象>_proposal.md` に保存

### CLAUDE.md の扱い
> ⚠️ `@import` の動作を PoC で確認してから確定する（現時点 TBD）

各スキルが展開する CLAUDE.md の設計は PoC 完了後に決定する。

## テスト方法

### How-1: テンプレートリポジトリ
| テスト | PASS 条件 |
|---|---|
| 新規リポジトリ生成 | `gh repo create --template` 後の構造が `モノレポ案.md` の仕様と差分ゼロ |

### How-2: テンプレート伝搬
| テスト | PASS 条件 |
|---|---|
| 同期スクリプト動作 | `/template-sync` 実行後、差分が意図した変更のみ |
| 既存ファイル保護 | 既存ファイルがある状態で同期を実行し、内容が変更されないこと |

### How-3: CLAUDE.md 分割
| テスト | PASS 条件 |
|---|---|
| @import 動作確認（PoC） | `docs/claude/*.md` の内容について問いかけ、Claude が正しく答えられること（ツール呼び出し結果で判断） |
| CLAUDE.md サイズ | `wc -l root/CLAUDE.md` の出力が設定値以下 |

### How-4: スキルによる操作
| テスト | PASS 条件 |
|---|---|
| `/monorepo-init` | 生成後の構造が `モノレポ案.md` の仕様と差分ゼロ |
| `/package-add` | `packages/<新パッケージ>/` がテンプレート通りに生成される |
| 別プロジェクトでの動作 | 全く別のリポジトリで `/monorepo-init` を実行して同じ結果が得られる |
| 既存ファイル保護 | 既存ファイルがある状態でスキルを実行し、内容が変更されないこと |

### Claude Code の出力先
| 出力の種類 | 正しい場所 | PASS 条件 |
|---|---|---|
| 新規アイデア | `ideas/ideas.md` | Claude が編集するファイルが `ideas/ideas.md` のみ |
| やること追加 | `tasks/backlog.md` | Claude が編集するファイルが `tasks/backlog.md` のみ |
| 横断的な計画変更 | `tasks/plan.md` | 同上 |
| パッケージ固有の計画 | `packages/<pkg>/plan.md` | 同上 |
| 設計判断ログ | `docs/decisions/XXXX-*.md` | 同上 |
| アーキテクチャ変更 | `docs/architecture.md` | 同上 |

### ネガティブテスト
| テスト | PASS 条件 |
|---|---|
| 「新しいアイデアがある」 | `ideas/ideas.md` に誘導され、`tasks/backlog.md` は編集されない |
| 「やることを追加したい」 | `tasks/backlog.md` に誘導され、`ideas/ideas.md` は編集されない |
