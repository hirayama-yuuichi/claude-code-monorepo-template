# アイデア管理

## アイデアの書き場所

思いついたアイデアはすべて `in-progress/ideas.md` に書く。
「まだやるか不明なもの」専用の置き場で、ここ以外に書かない。

## ideas.md から records への昇格フロー

アイデアを実際に進めると決めたとき、`/records-add` スキルで設計判断ログ（records）を作成する。

```
ideas.md にアイデアを書く
    ↓
やると決まったら /records-add でログを作成
    ↓
議論・判断（/critique → /propose → 採用）
    ↓
実装・docs 更新
    ↓
record のステータスを「完了」に更新
```

## 使用できるスキル

| スキル | トリガー | 用途 |
|---|---|---|
| `records-add` | `/records-add` | 設計判断ログを新規作成 |
| `critique` | `/critique` | ファイルを批判し `_review.md` を生成 |
| `propose` | `/propose` | `_review.md` をもとに修正案を生成 |
