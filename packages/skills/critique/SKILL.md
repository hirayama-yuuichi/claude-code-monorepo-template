# Critique — 批判スキル

## When to Use

Claude の提案や計画ファイルに対して、提案者バイアスのない批判的レビューを得たいとき。
採用ループ（critique → propose → 比較・採用）の第1ステップ。

## How It Works

1. ユーザーが `/critique <ファイルパス>` を実行する（パス省略時はカレントディレクトリの直近の変更ファイルを対象とする）
2. 対象ファイルを読み込む
3. 別インスタンス（サブエージェント）に以下の観点で批判させる：
   - Why/What/How の整合性
   - 実現可能性・具体性の欠如
   - テスト方法の妥当性
   - 計画として不足している情報
4. 批判結果を `<対象ファイル名>_review.md` として同じディレクトリに保存する
5. 保存先のパスをユーザーに通知する

## Output

- `<対象>_review.md` — 批判結果（一時ファイル。採用後は `_old/` へ移動）

## Notes

- 批判は別インスタンスで実行することでバイアスを排除する
- 次のステップは `/propose` で修正案を生成する
- 詳細: `decisions/0001-提案の採用ループ.md`
