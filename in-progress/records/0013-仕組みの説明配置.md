- 日付: 2026-04-27

### 背景

rec-0010 で整理した「やりたいこと」の3つ目。初見の人間・Claude の両方がすぐ理解できるように、テンプレートの目的・使い方の説明を適切な場所に集約する。最終的には packages を別リポジトリとして公開することを前提とする。

### 現状の説明の所在

| 内容 | 場所 |
|---|---|
| リポジトリ全体の目的 | `CLAUDE.md`（Claude 向け）、`README.md`（人間向け） |
| ディレクトリ構造の方針 | `docs/モノレポ案.md`（コメント形式で読みにくい） |
| スキル一覧・使い方 | `plan.md`（How セクション）に分散 |
| 採用ループ | `CLAUDE.md` と `docs/records/0001-提案の採用ループ.md` |

### 指摘

- `packages/template/README.md` は展開後に「このプロジェクトの README」として上書きされる想定のため、テンプレート説明を書く場所として適切かは要検討
- 別リポジトリ公開を前提にするなら、リポジトリの `README.md`（ルート）がテンプレートの説明を担う場所になる
- 展開先に「どのテンプレートから生成されたか」の参照行を `README.md` 冒頭に自動挿入する（`/monorepo-init` の仕様）のが現実的

### 判断

以下の2つの README を作成し、ターゲットを分離する：

1. **ルート `README.md`** — テンプレート開発者向け
   - このリポジトリの改善方法
   - 開発フロー（ideas → records → backlog → 実装）
   - スキルの管理方法

2. **`packages/README.md`** — テンプレート利用者向け
   - テンプレートの概要（何ができるか）
   - インストール方法（`/monorepo-init`）
   - 利用できる機能（スキル・ファイル構造）
   - 詳細なドキュメントへのリンク

### 理由

- ルート README とターゲットを分離することで、それぞれの読者に最適な情報を提供できる
- GitHub で `packages/` を開いたときに「使い方」がすぐ分かる
- ルート README は「このリポジトリをどう改善するか」に集中できる

### 却下した選択肢

| 案 | 却下理由 |
|---|---|
| ルート README に利用者向け情報も含める | 開発者向け情報と混在して読みにくい |
| `packages/README.md` を目次のみにする | 利用者が情報を探す手間が増える。先に削除した `packages/readme.md` と同じ問題 |
| `packages/template/README.md` に利用者向け情報を書く | 展開後に上書きされる想定のため不適切 |

### 影響

- ルート `README.md` を開発者向けに書き直す
- `packages/README.md` を新規作成（利用者向け）
- 両 README が相互にリンクする構造になる
