### 背景

`packages/template/docs/` には `architecture.md` と `setup.md` が存在するが、展開先プロジェクトで Claude Code と人間が「この仕組みをどう使うか」を理解できる説明が不足している。rec-0013（仕組みの説明配置）で検討すべき課題として挙げられていた内容。

なお、rec-0011（docs をテンプレート外に移動）の決定により、このリポジトリの docs は `packages/docs/` に配置される。

### 判断

`packages/docs/` に以下のファイルを追加する：

| ファイル | 内容 |
|---|---|
| `1_概要.md` | このテンプレートで利用できる仕組みの概要。1. モノレポ共有構造、2. アイデアを実際の形にするときの共通手順 |
| `2_モノレポ構造.md` | モノレポ構造概要（2ゾーン構造・ファイル配置方針） |
| `3_アイデア管理.md` | アイデアの管理（ideas.md の運用・records への昇格フロー） |

### 理由

- 構造と作業方法を分離することで、それぞれが読みやすくなる
- ファイルを番号付きにすることで読む順序が明確になる
- 段階的に理解を深められる構成（概要 → 構造 → 具体的な作業方法）

### 却下した選択肢

| 案 | 却下理由 |
|---|---|
| `architecture.md` 1ファイルにすべて記載 | 構造説明と作業方法が混在して読みにくい |
| `README.md` に記載 | 展開後に上書きされる想定のため不適切（rec-0013 参照） |
| `how-to-work.md` 1ファイルにまとめる | 概要・構造・アイデア管理を1ファイルに詰めると肥大化する |

### 影響
