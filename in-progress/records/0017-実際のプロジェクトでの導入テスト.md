# 設計判断ログ

## 0017: 実際のプロジェクトでの導入テスト

- 日付: 2026-05-02
- 状態: 議論中
- ドキュメント: 未着手
- 実装: 未着手

### 背景

このテンプレートリポジトリ（p030_claude_code_base）では以下が整備された：
- モノレポ構造（2ゾーン構造：docs/ と in-progress/）
- アイデア管理フロー（ideas → records → backlog → 実装）
- スキル群（/monorepo-init, /package-add, /records-add, /critique, /propose）
- 設計判断ログ（records）の運用

これらが実際のプロジェクトで有効に機能するか、実地でテストする必要がある。

### 判断

新規プロジェクトまたは既存プロジェクトでこのテンプレートを導入し、以下を検証する：

1. **初期セットアップの検証**
   - `/monorepo-init` で新規プロジェクトを立ち上げられるか
   - 展開後の構造が意図通りか
   - CLAUDE.md の指示が適切に機能するか

2. **日常運用の検証**
   - アイデアから実装までのフローが自然に回るか
   - records の採用ループ（/critique → /propose）が有効か
   - backlog の管理が負担にならないか

3. **スキルの実用性検証**
   - `/package-add` でパッケージ追加が円滑か
   - `/records-add` で設計判断が適切に記録されるか
   - `/critique` と `/propose` が実際の改善に役立つか

4. **問題点の洗い出し**
   - 使いにくい点、分かりにくい点
   - 不足している機能・ドキュメント
   - 過剰な構造・不要なファイル

### 理由

- テンプレート作成リポジトリ内では「メタな視点」が強く、実際の開発フローでの使い勝手が分からない
- 実プロジェクトで使うことで、構造の過不足や運用上の摩擦点が明確になる
- フィードバックをもとにテンプレートを改善し、実用性を高める

### 却下した選択肢

| 案 | 却下理由 |
|---|---|
| テンプレートリポジトリ内で模擬検証 | 実際の開発フローと異なるため、本当の使い勝手が分からない |
| ドキュメントレビューのみで完成とする | 机上では問題なくても、実運用で摩擦点が出る可能性が高い |

### 影響

- 新規プロジェクトまたは既存プロジェクトでの試用期間が必要
- 発見した問題点を records で記録し、テンプレートに反映する必要がある

### docs 記載方針

テスト完了後、以下を更新する可能性がある：
- `packages/docs/1_概要.md` — 導入時の注意点を追記
- `packages/docs/3_アイデア管理.md` — 運用上のコツを追記

### 実装方針

1. **テストプロジェクトの選定**
   - 新規プロジェクト、または構造を再整理したい既存プロジェクトを選ぶ

2. **テスト手順**
   ```bash
   # 新規プロジェクトの場合
   mkdir test-project && cd test-project
   # Claude Code で /monorepo-init を実行
   
   # 既存プロジェクトの場合
   # テンプレートの一部（in-progress/、スキル）を手動で導入
   ```

3. **検証期間**
   - 最低1週間、実際の開発タスクをこのテンプレート上で実行
   - 気づいた点は随時 ideas.md または新しい records に記録

4. **フィードバック収集**
   - 使いにくかった点
   - 改善案
   - 不要だったファイル・構造

5. **テンプレートへの反映**
   - 発見した問題を records 化
   - 必要に応じて構造・スキル・ドキュメントを修正
