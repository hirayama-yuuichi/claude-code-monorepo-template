# 設計判断ログ

## 0003: スキルのテンプレート参照

- 日付: 2026-04-25
- 状態: 採用
- [ ] docs/ 更新済み
- [ ] 実装済み

### 背景

`/package-add` スキルは `packages/template/packages/package-a/` というローカルパスをテンプレートとして参照している。
一方 `/monorepo-init` は `degit` URL（GitHub 経由）を使っており、両者で参照方式が異なる。

`.claude/skills/` に置かれたスキルが別プロジェクトにコピーされた場合、`packages/template/` は存在しないため `/package-add` が壊れる。

### 選択肢

| 案 | 内容 | メリット | デメリット |
|---|---|---|---|
| A | `/package-add` も `degit` ベースにする | どこでも動く | ネット必須・毎回 fetch が走る・遅い |
| B | テンプレートパスを設定ファイルから読む | 柔軟 | 設定ファイル管理が増える |
| C | スキルはこのリポジトリ専用と割り切り、プロジェクトローカル配置のみとする | 今の方針と整合・シンプル | 他プロジェクト転用時に要改修 |

### 決定

**A案（スキルを自己完結させ degit ベースに統一する）**

スキルは最終的に別プロジェクトの `.claude/skills/` にコピーして使うことを想定している。
`packages/template/` は仕様の方向性確認用であり、スキルの実行時依存にしない。
スキルは自己完結させ、テンプレートが必要な場合は GitHub 経由（degit）で取得する。

`/monorepo-init` はすでにこの形になっている。`/package-add` のローカルパス参照を degit に修正する。

```
このリポジトリ
├── packages/template/  ← 仕様の実体（GitHub に push → degit の取得元）
└── packages/skills/    ← スキルのソース（自己完結・他プロジェクトに配布）

別プロジェクト
└── .claude/skills/     ← コピーされたスキル（degit で GitHub 参照）
```

### 影響

- `/package-add` のテンプレート取得を `packages/template/` ローカル参照から degit に変更する
- `/template-sync`（未作成）も同様に degit ベースで設計する
- `packages/template/` の変更は GitHub に push することでスキルに反映される
