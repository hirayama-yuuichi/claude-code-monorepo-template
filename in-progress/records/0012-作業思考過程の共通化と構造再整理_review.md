# レビュー：0012-作業思考過程の共通化と構造再整理

## 1. Why/What/How の整合性の問題

- **背景が不十分**：「共通化のルールが曖昧」と指摘するのみで、「何がどう曖昧か」「現在の問題が何を招いているか」が記載されていない。判断に至る動機が不明確
- **「判断」が先走っている**：背景で「曖昧さを明確にしたい」と述べながら、「指摘」セクションが空。判断の根拠となる問題分析がない

## 2. 実現可能性・具体性の欠如

- **実装対象が曖昧**：「`packages/docs/3_アイデア管理.md` に整理する」とあるが、具体的に何を書くのか（既存との差分は？）が不明
- **CLAUDE.md への影響が漠然**：「出力ルール表がこの構造を反映している」と述べるが、具体的にどこをどう反映すべきか記載なし

## 3. 計画として不足している情報

- **records/ の長期運用が不明**：「採用後もここに置いたまま」とあるが、増え続けた場合の退避ルールがない
- **役割分担の実運用ルールがない**：「Claude が管理」「人間が管理」と述べるが、更新タイミングや編集ルールがない

## 4. 内容の矛盾や重複

- **「省略」セクション**：「現状の構造」が「省略」のまま。削除予定かどうか不明
- **判断と実装方針の重複**：「判断」で構造を定義し、「実装方針」で docs に記載するとあるが、docs 記載方針が「実装とかぶるので不要」なら実装方針も同様の問題を抱える
