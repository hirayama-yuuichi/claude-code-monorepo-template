### 背景

rec-0010 で整理した「やりたいこと」の1つ目。現状の `packages/template/` 構造を確認し、展開先プロジェクトで即日機能する状態にするための不足・ズレを明らかにする。

### 現状の構造（template/）

```
packages/template/
├── README.md                        ← リポジトリ入口。プロジェクト概要・パッケージ一覧・最短の使い方
├── docs/
│   ├── architecture.md              ← 全体構成・パッケージ間依存・責務分離方針
│   ├── setup.md                     ← 環境構築手順（新人がここだけ見れば動く状態を目指す）
│   └── records/
│       └── 0001-xxx.md              ← 設計判断ログ（ADR）のプレースホルダー
├── ideas/
│   └── ideas.md                     ← まだやるか不明なアイデアの一元管理
├── tasks/
│   ├── backlog.md                   ← やること一覧（唯一のTODO。ここだけに書く）
│   └── plan.md                      ← パッケージ横断の計画（Why/What/How）
└── packages/
    ├── package-a/                   ← サンプルパッケージ（実装はここに閉じる）
    │   ├── README.md                ← このパッケージの入口（何をするか・使い方・依存関係）
    │   ├── plan.md                  ← このパッケージ内の計画（横断の話は tasks/plan.md へ）
    │   ├── setup.md                 ← このパッケージ単体の開発手順
    │   └── spec/
    │       ├── api.md               ← API仕様
    │       ├── data.md              ← データ構造・スキーマ
    │       └── rules.md             ← ビジネスルール・制約
    └── package-b/
        └── .gitkeep                 ← 空パッケージのサンプル
```

### 指摘

- `CLAUDE.md` がない → 展開先プロジェクトで Claude Code が出力ルール・採用ループを知れない
- `docs/records/0001-xxx.md` がプレースホルダーのみ → ADR の書き方・用途が伝わらない
- `tasks/plan.md` はあるがルートの `plan.md` はルート直下にあり非対称（rec-0009）
- `docs/setup.md` の内容が空に近く、展開後に使えない状態
